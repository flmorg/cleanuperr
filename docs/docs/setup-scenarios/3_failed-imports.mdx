# Managing Failed Imports

Automatically handle downloads that fail to import properly into your *arr applications using the Queue Cleaner's strike system.

<div className={styles.documentationPage}>

<div className={styles.section}>

<StepGuide>
  <Step title="Enable Queue Cleaner">
    Use **Cleanuparr** with `Queue Cleaner` enabled.
  </Step>

  <Step title="Configure Strike System">
    Configure `Failed Import Max Strikes` to set how many failed attempts before removal.
  </Step>

  <Step title="Automated Processing">
    The **Queue Cleaner** will monitor import failures and apply strikes as described in the [How it works](/docs/how_it_works) section.
  </Step>
</StepGuide>

<EnhancedNote>
This scenario focuses on handling import failures. Other features can be combined by configuring additional Queue Cleaner settings.
</EnhancedNote>
